About

"Coffeeverse" was my final project in my studies for Frontend developer at IT-HÃ¶gskolan in order to get my degree. It is a web app where one goes to order coffee and other snacks for "fika" and can choose among different coffee shops in their city. Right now it is functioning in a basic, demonstrative level and the user cannot place an actual order, but rather a fictional one. It was an idea I have had for a long time so that was the perfect instance to implement it!

Requirements

First, I conducted reasearch with some users in order to determine the relevance of the web app as well as most important features. Research was conducted first with a questionnaire and then with two more in-depth interviews.

Basic requirements for the project:

  • A variety of drinks and snacks to choose from
  • A variety of coffee shops
  • Filtering and sorting functions for a user-friendly website
  • See product ratings
  • Visible product prices and offers/ discounts
  • User is able to review their order status: Time left to delivery, items in order
  • Complete order by entering delivery info and paying (not real payment)
  • Two options for payment: credit card or swish
  • Feedback to user when order is placed successfully
  • User friendly design according to UX guidelines
  • Responsive website adjusted to mobile and tablet screens
  • App in English

As the project was realized within three weeks, some of the basic requirements were not fulfilled. Functions that were important for the website to work were prioritized over enhancement features, but it is definitely a work in progress that will be much improved in the future.

Technology

In order to create the lo-fi and hi-fi prototypes before programming, I used Figma . Images were first edited with GIMP to remove unnecessary background and to fix dimensions. The final versions of the images used in the website were created with Figma in order to add the brand icon.

Firebase Firestore was used for storing data about the products in the "Coffeeverse" and fetching them via API in my Vue application. It was also used to store all the images, as this minimized the storage my app was going to occupy

Frontend development of the web app was done with Vue js and Boostrap-Vue. Focus was on organizing the Vue components in an efficient way, applying the DRY (Don't repeat yourself) principle and translating the UI design to code, in order to deliver the best experience to the user.
